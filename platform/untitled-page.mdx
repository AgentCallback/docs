## Production Prompt Framework:

A production-ready voice agent should be organized into clearly defined sections.

<Steps>
  <Step title="Agent Identity.">
    Define who the agent is and its role. 

    <sub>Ex: You are Eva from {{Clinic\_name}}. Your role is to help customers schedule consultations and -</sub><br /><sub>answer appointment-related questions.</sub>
  </Step>
  <Step title="Business Objectives">
    Define what the agent should accomplish.

    <sub>Ex: - Book appointments.  - Collect callback requests when needed.  - Escalate complex questions to a human agent.</sub>
  </Step>
  <Step title="Conversation Style">
    Define tone, personality, and speaking style.

    <sub>Ex:  - Keep responses short and natural.  - Ask only one question at a time. </sub>
  </Step>
  <Step title="Conversation Rules">
    Define global rules the agent must follow.

    <sub>Ex:  - Never interrupt the customer. - If the customer says "Not interested", politely end the conversation. - Confirm important details before proceeding</sub>
  </Step>
  <Step title="Conversation Flow">
    Define how the conversation should progress.

    <sub>1. Greet the customer. 2. Verify their identity. 3. Explain the reason for the call. 4. Collect the required information. 5. Complete the booking. 6. End the call politely.</sub>
  </Step>
  <Step title="Tool Instructions">
    Define when and how external systems should be used.

    <sub>Ex: Before offering an appointment, call the </sub>`checkCalendarAvailability`<sub> tool. Only offer time slots returned by the tool. Never create appointment times from memory.</sub>
  </Step>
  <Step title="Exception Handling">
    <sub>Ex: If the customer asks about any specific medicines, do not answer. Instead, explain that a specialist will contact them with accurate information.</sub>
  </Step>
  <Step title="Termination Rules">
    Define when the conversation should end.

    <sub>End the conversation after: - Appointment is booked. - Customer declines the service. - Wrong person is reached. - Customer requests no further contact.</sub>
  </Step>
  <Step title="Outcome Classification">
    Define how the final result should be classified.

    <sub>BOOKED\_APPOINTMENT - Appointment successfully scheduled. </sub>

    <sub>REQUESTED\_CALLBACK - Customer requested a callback. </sub>

    <sub>NOT\_INTERESTED - Customer declined the offer.</sub>
  </Step>
</Steps>

## General Guidance

#### Start with a Basic Prompt

Begin with a simple prompt that captures the core behavior you want from the agent. Test it first, then gradually add more instructions, rules, or edge cases as you refine the prompt.

#### Write Clear Instructions

Be specific about what you expect the model to do. Clear, unambiguous instructions help produce more reliable responses. When defining workflows, break them into logical and easy-to-follow steps.

#### Include Examples

Providing sample interactions or expected outputs helps the model better understand the desired behavior. A few well-written examples can significantly improve consistency.

#### Refine Through Testing

Prompt design is an iterative process. Test your prompt, review the agent's responses, identify areas for improvement, and update the prompt accordingly. Repeating this cycle leads to more accurate and dependable results.

Start your prompt by providing the agent with any dynamic information it may need during the conversation.

During runtime, AgentCallback replaces these variables with real values before the conversation begins, enabling the agent to make accurate decisions based on the current context. These common variables available by default.

| {{currentDay}} | Current weekday (defaults to Los Angeles). Supports any IANA timezone. |
| {{currentTime}} | Current time (defaults to Los Angeles). Supports any IANA timezone. |

You can create your own variables and reference them anywhere in your prompt using the`{{variable_name}}` format. Custom variable values can be supplied from a variety of data sources, including CSV imports, API responses, CRM integrations, contact lists, and workflow data.

## 3. Workflow Branching

Most voice conversations are not linear. Customers may accept your offer, decline it, ask questions, request more information, or simply be unsure about the next step. 

To keep conversations predictable, your prompt should clearly define how the agent should respond to each possible customer intent.

Example:

```text
Would you like to schedule your consultation now?

If the customer agrees(e.x. Yes, I'm Available): → Continue to the 'Booking Flow'
If the customer declines(e.x. No): → Follow the 'Not Interested' workflow
If the customer is unsure or hesitant(e.x. May be): → Offer a 'Specialist Follow-Up'
If the customer asks a question(e.x. Tell me more): → Follow the 'Out-of-Scope Handling process'
```

Explicit branching reduces unpredictable behavior and improves task completion rates.

## 4. Tool Instructions

Voice agents can access external systems via a [<u>Tool</u>](https://app.mintlify.com/agentcallback/agentcallback/editor/main/~/8835bbd0-9a1e-440c-b18f-653db59cdbf6) call such as calendars, CRMs, databases, and APIs to retrieve information or perform actions during a conversation.

Provide clear instructions in the prompt on when the agent should retrieve information and how it should use the results.

Example:

```text

- Before scheduling an appointment: 
  → Call "gcCheckCalendarAvailability" tool to verify the timeslots
- Offer only available slots.
- If the requested time is unavailable, offer an alternative.
```

Clear instructions help the agent use tools correctly and consistently.

## 7. Date and Number Handling

Voice agents should convert dates, times, and numbers into natural spoken language. This makes conversations easier to understand and creates a more human-like experience.

Instruction Example:

```text
### Pronunciation Guide
* Dates in DD-MM-YYYY format should be spoken naturally.
 → Example: 09-06-2026 → "9th June 2026".
* Dates in DD/MM/YYYY format should be spoken naturally. 
 → Example: 31/05/2026 → "31st June 2026".
* Times should be spoken conversationally. 
 → Examples: 10:00 AM → "Ten AM", 10:30 P M → "Ten thirty P M".
```

## 8. Optimize Token Consumption

AI agents consume tokens while processing prompts and generating responses. Well-structured prompts not only improve accuracy but can also reduce unnecessary token usage.

Follow these best practices when writing prompts:

> ### Keep Instructions Clear
>
> Use simple, direct instructions whenever possible.
>
> **Avoid :**
>
> Large separators and decorative formatting increase prompt size but rarely improve agent performance.
>
> ```text
> ====================
> GREETING
> ====================
> ```
>
> **Recommended :**
>
> ```text
> ## Greeting
> ** Greeting **
> ```
>
> ### Avoid Repeating Instructions :
>
> State a rule once and keep it concise.
>
> **Avoid**
>
> ```text
> Never offer past appointment times.
>
> Do not schedule appointments in the past.
>
> Only offer future appointments.
> ```
>
> **Recommended**
>
> ```text
> Never offer past appointment times.
> ```

# 10. Outcome Classification

Every production voice agent should generate a clear outcome at the end of each conversation. Outcome classification allows you to measure performance, automate follow-up actions, and gain visibility into how your campaigns and workflows are performing.

By defining outcomes upfront, the agent can consistently categorize conversations based on the final result rather than relying on manual review.

```text
Common examples include:

BOOKED_APPOINTMENT

REQUESTED_CALLBACK

NOT_INTERESTED

WRONG_PERSON

VOICE_MAIL

CALL_DISCONNECTED
```

```text Mandotory Instruction to add in the Outcome Prompt:

At the end of a call, 
AgentCallback can evaluate the conversation using information such as:

{{transcript}}

{{shortSummary}}

{{endReason}}

The outcome prompt analyzes this information and returns a single outcome based on the rules you define.
```

```text
Example: 

The following example returns BOOKED_APPOINTMENT 
when an appointment has been successfully scheduled.

Call Transcript:
{{transcript}}

Call Summary:
{{shortSummary}}

End Reason:
{{endReason}}

If the transcript or summary indicates that:

- An appointment was successfully booked
- The customer confirmed the appointment
- The scheduling tool created the appointment

Return exactly:

BOOKED_APPOINTMENT
```

## Outcome Order

### Definition

When multiple outcome conditions could match the same conversation, the order of your outcome rules determines which outcome is returned.

AgentCallback evaluates outcomes from top to bottom and returns the first matching outcome.

### Why It Matters

A single conversation may satisfy more than one outcome condition.

For example, a customer might initially decline the offer but later agree to book an appointment. If `NOT_INTERESTED` is checked before `BOOKED_APPOINTMENT`, the conversation could be classified incorrectly.

<Tip>
  Always place your most important and specific outcomes first.
</Tip>

**Recommended:**

> - BOOKED\_APPOINTMENT
> - REQUESTED\_CALLBACK
> - NOT\_INTERESTED
