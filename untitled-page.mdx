### What Are Tools?

Tools allow your voice agent to interact with external systems during a conversation.

Without tools, an agent can only rely on the information available in the prompt. With tools, the agent can retrieve information, update records, and perform actions in real time.

Common examples include:

- Checking calendar availability
- Creating appointments
- Looking up customer records
- Sending SMS messages
- Updating CRM systems
- Triggering business workflows

### How AgentCallback Uses Tools

AgentCallback allows you to connect APIs, databases, CRMs, calendars, and other external services to your voice agents.

When the agent needs information or needs to perform an action, it can automatically use the appropriate tool and continue the conversation using the returned result.

For example, during an appointment scheduling call:

1. The customer asks for an available appointment.
2. The agent checks the connected calendar.
3. Available time slots are returned.
4. The agent offers those slots to the customer.
5. Once confirmed, the agent creates the appointment.

All of this happens automatically during the conversation.

### Example

Prompt:

```text
Before scheduling an appointment, check the calendar for available time slots.

Offer only available slots.

Never make up appointment times.
```

Tool:

```json
{
  "name": "gcCheckCalendarAvailability",
  "description": "Returns available appointment slots."
}
```

The agent will automatically use the tool whenever appointment availability is needed.

### Best Practices

- Clearly define when a tool should be used.
- Instruct the agent how to use the tool results.
- Never allow the agent to invent information that should come from a tool.
- Keep business logic in the prompt and authentication in your backend or integration layer.
